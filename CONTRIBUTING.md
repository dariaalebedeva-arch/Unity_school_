# Contributing Guide

Спасибо за интерес к проекту Unity School Landing.

## Как предложить изменения

1. Сделайте форк репозитория (или создайте ветку в текущем репо).
2. Создайте ветку под задачу:
   - `feat/<short-name>` для фич
   - `fix/<short-name>` для исправлений
   - `docs/<short-name>` для документации
3. Внесите изменения небольшими логическими коммитами.
4. Проверьте, что сайт корректно открывается локально.
5. Оформите Pull Request с кратким описанием:
   - что изменено,
   - зачем изменено,
   - как проверить.

## Стандарты кода

- Используйте понятные имена классов и функций.
- Сохраняйте единый стиль форматирования.
- Не добавляйте неиспользуемый код и закомментированные блоки “на будущее”.
- Держите изменения фокусными и минимальными.

## Проверка перед PR

- `index.html` открывается без ошибок.
- Новые элементы адаптивны (мобильный + десктоп).
- Анимации не ломают UX и учитывают `prefers-reduced-motion`.
- Нет битых ссылок и опечаток в контенте.

## Коммиты (рекомендуемый формат)

- `feat: add new testimonials layout`
- `fix: correct phone validation in application form`
- `docs: update readme with deployment notes`
- `style: refine hero spacing on mobile`

## Вопросы и предложения

Если хотите предложить крупное изменение (новый раздел, редизайн, интеграцию backend),
лучше сначала открыть issue с коротким описанием идеи и ожидаемым результатом.
